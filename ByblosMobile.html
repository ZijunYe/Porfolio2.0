The project of class introduction of software engineering was done by a team of 5 people. This application implements some basic functionalities of services offered by a car rental company called Byblos to its residents.

Overview & Mapping

The app has three types of user: administrator, employee(service provider), and user. Based on the project description, we build the system domain model to help us to understand the relationship. The class diagram has limited user interface and mainly focuses on the functionality.

Functionality & Demo

Login/Signup

User able to create their account by choose to be Employee or customer, but the administrator account is preset account. Also when user login to their account, user need to select their role. All input section has the validation; once the password matched with the database record, user are able to see the welcome page with their username.

User#1: Administrator

The features of administrator section are: able to create service, remove service, and edit service. The administrator also able to delete all exist customer and employee account.

User#2: Service Provider

We assume that service provider as branch; only one account per branch. The features of service provider section are: able to select services that provided(from set of all available service), select the service working hours of the branch, and edit the details information of the branch(address,email,branch name,phone number). Once the branch received the requests from customer, service provider are able to view the requests, and either approve or reject each requests.

User#3: Customer

The main feature of customer section is able search service by address/type of service provided/working hour. If customer decided to make purchase, they able to select the target branch and fill the required information to make service request. After the request done, customer also able to rate and give feed for the branch.
